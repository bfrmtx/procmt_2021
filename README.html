ProcMT 2021

ProcMT stands for Process MagnetoTelluric data.

It is your QC (quality control) tool for your MT data.
Your professional SW should give better results. If ProcMT gives no results, it is questionable if your data is good enough for professional processing.

I was not able to make a complete port of the “older” ProcMT version. But most - and all important - parts are ported to Qt 6.X

May 2023: changes for Qt 6.5

August 2023: edi files with ZROT and TROT (required by Geotools®)

December 2023: Mac OS port.

June 2026: split and unsplit.

The induction arrows seem not to be good.

Prerequisites

  • Linux: clang++ / gcc compiler with C++ 20 support
  • Apple clang version 12 or above
  • tested with MinGW64 compiler & Ninja make, shipped with Qt 6.5
  • CMake 3.24 or above (Version is upgrading fast - Cmake is tied to compilers and their features)
  • C++ Boost library installed 1.6 or later
  • the Boost libraries do not necessarily to be compiled
  • Qt 6.5 or later be installed (on Mac OS: brew install qt)

Optional

  • on Windows Khronos Vulkan is needed in order to avoid error messages Vulkan
  • C++ package Qt 6.5 or later
  • OpenGL and Sql libraries from Qt

START

If you are smart - you simply use my settings and do not need to edit anything.
Especially if you run updates from github, you do not need edit your files again and again.

You always need a install directory, where the executable is copied to.
YOU must have read/write access to this directory.

sudo mkdir -p /usr/local/procmt
# Linux:
sudo chown -R $USER:$USER /usr/local/procmt
# Mac OS:
sudo chown -R $USER:staff /usr/local/procmt

When later rebuilding the project, you can use the same directory again.
Simply go INTO the directory and delete all files and subdirectories.
DO NOT delete the directory itself. For the professionals:

cd /usr/local/procmt
# check with pwd that you are there!
pwd
# and then delete all files and subdirectories
rm -rf *

otherwise use your file manager!

In case you want to save time, use this:

cd $HOME
mkdir -p $HOME/build
mkdir -p $HOME/devel/github_procmt_2021
cd $HOME/devel/github_procmt_2021
git clone https://github.com/bfrmtx/procmt_2021.git
cd procmt_2021
zsh cpp_build_release.zsh

On Windows, after installing Qt and MSCV 2022:

You can use my .bat files, located in windows_bat directory.
You need to edit Path, username, and maybe the Qt version.
Then you can use the .bat files to build the project.
I open “Windows PowerShell for VS 2022” or “Developer Command Prompt for VS 2022” and run the .bat files from there. When the Terminal opens, you get a message:
** Visual Studio 2022 Developer Command Prompt v17.7.4 **
(or similar, depending on your version)
Then you can run the .bat files.

Rebuild

When forcing a rebuild, you simply delete the build directory and start again.

cd $HOME/build/procmt_2021
rm -rf *

Version 2025

The filter delay is shifted by one point.

So for a 71 length FIR filter at 512 Hz I shift 477 points
(old version was 476 points)
Simply filter your source runs again if you want to be compatible in future.
